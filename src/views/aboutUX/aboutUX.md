# 前端交互建议

## 列表页

- 经典布局：上搜索栏，中间列表、下翻页器
- 搜索栏、翻页器固定，只有表格可滚动
- 搜索栏的搜索条件可折叠展开
- 搜索、翻页需加入 loading 效果
- 表格空数据效果
- 列表左边的“序号”列固定，表格右边的“操作”列固定
- “操作”列的按钮颜色使用。普通功能（蓝色），删除（红色）
- “操作”列建议使用左对齐或右对齐，不要居中
- “操作”列，删除按钮必须有二次确认交互

## 批量操作按钮栏

- 搜索栏、列表的中间可以加入操作按钮栏，通常放置批量操作按钮，例如：批量删除

## 搜索栏

- 使用 el-form，有快捷的检验、重置数据功能。避免自己写校验逻辑
- 各个搜索条件加入一键清除功能
- 建议使用按钮（搜索、重置）主动触发搜索，避免搜索条件自动触发搜索，增加服务器负担
- 如使用搜索条件自动触发，应当加入函数节流放置频繁触发异步请求
- 按钮（搜索、重置）加入 loading 效果
- 🤔建议不要启用 el-form-item 的 label 属性，使用 placeholder 代替，节省更多空间
- 如果存在搜索条件联动，不要使用自动触发搜索，增加服务器负担

## 翻页器

- 建议向右对齐

## 弹窗 dialog

- 少内容可使用，例如：表单填写内容 <= 5个
- 展示数据无输入操作，可加入点击遮罩层关闭功能
- 有输入操作，取消点击遮罩层关闭功能

## 抽屉 drawer

- 多内容可使用，例如：表单填写内容 >= 5个
- 展示数据无输入操作，可加入点击遮罩层关闭功能
- 有输入操作，取消点击遮罩层关闭功能

## 页面使用说明

- 🤔建议在批量操作按钮栏加入“使用说明”按钮，提供业务说明、操作提示，对用户友好

## 使用习惯及操作直觉

- 搜索栏、批量操作按钮栏、翻页器，统一右对齐。因为表格的操作列靠右固定，视觉中心、交互直觉在右方
- “删除”操作都需要有二次确认交互
